## The Six Permutations

| Sequence | Leads with | Use case |
|---|---|---|
| **AID** | Abductive / Form | Default. Read the artifact before naming it. Teacher documentation, CFA panels, classroom observation. |
| **IAD** | Inductive / Function | Pattern already visible. Track recurrence first, then read backward, then test. Longitudinal data review. |
| **DAI** | Deductive / Process | Standard known. Design the test first, then observe what it reveals, then read the artifact backward. High-stakes assessment design. |
| **ADI** | Abductive → Deductive → Inductive | Read the artifact, design the response, then track whether the pattern holds over time. Intervention design. |
| **IDA** | Inductive → Deductive → Abductive | Pattern identified, test designed, then read what the test actually showed. Post-assessment reflection. |
| **DIA** | Deductive → Inductive → Abductive | Compliance-first contexts. Standard drives the design, pattern emerges in delivery, artifact read at close. CalTPA / CSTP documentation. |

---

## Architectural implication

The AID loop is not a fixed sequence. It is a set of three moves that can be ordered by context.

The question is always: **what is the entry point for this specific situation?**

- If you have an artifact: start Abductive (AID)
- If you have a pattern across time: start Inductive (IAD)
- If you have a standard or mandate: start Deductive (DAI or DIA)
