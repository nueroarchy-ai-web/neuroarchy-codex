## On the Paper-Writing Interface

"If the goal is to write a paper, I need to see the structure of the paper inline or on the side."

This is the Obsidian instinct made explicit. Not a blank page. Not a rubric handed down.
A visible structure — sidebar or inline — that holds the skeleton while Scott fills the body.

The AI-guided form already does this progressively.
The paper-writing version does it spatially:
- Left: the structure (outline, sections, headers)
- Right: the writing field for the current section
- Below each section: one AI prompt reading what was just written, offering the bridge to the next

This is Form/Function/Process applied to writing:
- **Form** — see the shape of the whole paper before writing any of it
- **Function** — write into each section knowing where it sits in the whole
- **Process** — the paper that emerges is the demonstration, not the proof

"We just need a form, function and process for us to produce the same for others."

The tool Scott uses to write is the same tool Scott's students use to think.
The architecture does not change — only the level of abstraction.
