## The Core Insight

The IB unit is not a separate framework bolted onto the panel tool.
It is the generative engine for what you are looking for in the artifact.

**Expansion chain:**
Line of inquiry → abductive prompt → observation block → panel

---

## IB Inquiry Cycle mapped to Form/Function/Process

| IB Phase | AID | Scott's Term | What it does |
|---|---|---|---|
| Tuning in | Abductive | Form | What is the shape of this before I name it? Pure receive. |
| Finding out | Inductive | Function | What pattern is emerging across the evidence? |
| Sorting out | Inductive | Function | What recurs? What is stabilizing? |
| Going further | Deductive | Process | What is the fair demonstration of the learning? |
| Taking action | Deductive | Process | Apply the learning in a new context |
| Reflecting | Abductive | Form | Read backward — what does this artifact suggest about what the learner now holds? |

The inquiry cycle and the AID loop share the same underlying move: receive before categorize.

## The Three-Layer Nested Structure

**Layer 1 — Hard Frame (filter/settings)**
Set once per session. Stamps everything.
- Standards: CEL, CAASPP, CalTPA focus, CSTP element
- DOK target (Webb's 1–4)
- 5x5 stage
- IB transdisciplinary theme
- Learner profile target

**Layer 2 — Lesson Skeleton**
- IB unit title
- Central idea
- Lines of inquiry (these are the expansion logic — each line generates an abductive prompt)

**Layer 3 — Panel**
- Artifact
- Observation (present tense, low inference)
- Abductive read — BUT NOW: the prompt for the abductive read is generated from the line of inquiry above it
- Tags (AID · 5x5 stage · ZoT layer · CEL standard)
