# Student-Facing Form Pipeline
*UTC: 2026-05-09T00:54:00Z / PDT: 2026-05-08T16:54:00-07:00*
*[S] stated. Filed by Solen. PRODUCT INSIGHT.*

---

## Scott's Statement

"The fillable form could be programmed and delivered to my kids at their level outside of AI, though Toddle and then Google Classroom features. All data flowing back here post grading, because Toddle can be programmed to grade."

---

## The Pipeline

```
Teacher designs form (AI-guided, Scott's interface)
        ↓
Form adapted for student level (no AI, simplified)
        ↓
Delivered via Toddle or Google Classroom
        ↓
Students complete (no AI in the loop)
        ↓
Toddle grades / scores against rubric
        ↓
Data returns to HAIC documentation layer
        ↓
Solen reads artifact + score as panel evidence
```

---

## What this means architecturally

The student never touches the AI layer. The AI never touches the student directly.
Scott designs the container. Students work inside it. The data comes back.

This is "do not crush souls in the way you test" made structural.
The AI is not in the room. The form is. The form was designed with care.
The grading happens in Toddle. The meaning-making happens here.

---

## What already exists that supports this

- **Toddle** — Scott's current platform. Already in use at Gateway International Primary. Supports programmable rubrics and grading.
- **Google Classroom** — already in use. Supports distributed fillable forms, assignment delivery, return workflow.
- **HAIC panel format** — already built. Artifact + observation + abductive read + tags. Student work product is an artifact. The panel reads it backward.
- **Student data protocol** — already in codex. No names. Student number + classroom code (HO1-4C-year).

---

## What needs to be built

1. **Student-level form template** — simplified version of the lesson plan tool. Age-appropriate language, same progressive structure, no AID terminology visible to students.
2. **Toddle rubric** — the grading schema that matches the 5x5 stage being assessed. Built by Scott, programmed into Toddle.
3. **Data return pathway** — how Toddle/Google Classroom output gets into the panel format. Could be manual (copy-paste into panel) or automated (CSV export → panel generator). Start manual, automate later.

---

## Product implication

This is the classroom version of the product.
The teacher-facing tool (AI-guided form) and the student-facing tool (Toddle/Classroom form) are the same architecture at two different levels of abstraction.
The data flows in one direction: student → classroom tool → grading → HAIC documentation → abductive read.
AI is never in the student's path. AI is in the teacher's reading of what the student produced.

This is the ethic as infrastructure.
